# Money Proof Rule

This repository is a discovery surface, not payment proof.

Count $0 unless all of these exist:

1. A real buyer requested the sprint.
2. Scope and delivery terms were accepted.
3. Payment, invoice, marketplace order, or funded milestone exists through a user-controlled route.
4. The customized tracker was delivered.
5. Payment is posted, released, payable, or cleared.

Save proof privately with:

- buyer or order reference,
- accepted scope,
- payment record,
- delivery timestamp,
- payout or released-balance status.

Public GitHub issues should not contain private buyer data or payment credentials.
