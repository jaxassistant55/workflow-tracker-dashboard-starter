## Payment Activation

After the buyer accepts the fixed scope, open:

https://jaxassistant55.github.io/jax-micro-offer-studio/payment-activation

Paste a seller-owned checkout, invoice, marketplace order, funded milestone, or payment request URL. The generated message is only a payment handoff. Work starts only after external payment proof exists.

## Buyer Inputs Needed

- Workflow type.
- Required columns.
- Desired statuses.
- Up to 25 starter rows or sample notes.
- Any fields that should be hidden, excluded, or renamed.
- Delivery preference for the final ZIP.

## Acceptance Checklist

- Buyer confirms all supplied inputs are authorized for use.
- Buyer accepts the fixed $150 scope.
- Buyer uses an external payment route controlled by the seller.
- Work starts only after payment or funded milestone is authorized.
- Final delivery includes the CSV, HTML dashboard, and handoff note.

Do not post private data, payment screenshots, API keys, customer records, or sensitive files in GitHub issues.
